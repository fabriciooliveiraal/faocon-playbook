# Padrão de Consultoria FAOCON

## Objetivo

Este documento define o padrão base para projetos de consultoria empresarial da FAOCON.

Todo projeto deve buscar clareza, organização, decisão e execução.

## Etapas principais

## 1. Entrada do projeto

Coletar:

- Contrato.
- Escopo.
- Objetivos.
- Prazos.
- Stakeholders.
- Critérios de sucesso.
- Limitações.
- Riscos iniciais.

## 2. Diagnóstico

Analisar:

- Modelo de negócio.
- Estrutura organizacional.
- Comercial.
- Financeiro.
- Operação.
- Pessoas.
- Processos.
- Tecnologia.
- Indicadores.
- Gargalos.
- Oportunidades.

## 3. Análise

Organizar achados em:

- Problemas críticos.
- Causas prováveis.
- Impactos financeiros.
- Impactos comerciais.
- Impactos operacionais.
- Riscos.
- Potenciais ganhos.

## 4. Recomendação

Toda recomendação deve conter:

- O que fazer.
- Por que fazer.
- Como fazer.
- Quem deve fazer.
- Quando fazer.
- Qual indicador acompanhar.
- Qual risco observar.

## 5. Plano de ação

Padrão mínimo:

- Ação.
- Responsável.
- Prazo.
- Status.
- Prioridade.
- Indicador.
- Observação.

## 6. Indicadores

Priorizar indicadores executivos:

- Receita.
- Margem.
- EBITDA quando aplicável.
- Fluxo de caixa.
- Inadimplência.
- Conversão comercial.
- Ticket médio.
- Carteira ativa.
- Produtividade.
- Cumprimento do plano de ação.

## 7. Relatórios

Relatórios devem conter:

1. Resumo executivo.
2. Situação atual.
3. Principais achados.
4. Indicadores.
5. Riscos.
6. Recomendações.
7. Plano de ação.
8. Próximos passos.

## 8. Padrão visual

- Linguagem executiva.
- Visual limpo.
- Hierarquia clara de informação.
- Tabelas objetivas.
- Gráficos quando agregarem clareza.
- Evitar excesso de texto em telas executivas.

## Diretriz para o Codex

Ao criar qualquer ferramenta, dashboard, relatório, sistema ou documento de consultoria FAOCON, seguir este padrão como base.
