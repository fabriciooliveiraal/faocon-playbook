# CODEx Instructions — FAOCON

Antes de iniciar qualquer desenvolvimento:

1. Ler todos os arquivos da pasta docs/contexto.
2. Entender o problema de negócio antes de propor tecnologia.
3. Priorizar simplicidade operacional.
4. Priorizar indicadores executivos.
5. Explicar a arquitetura antes da implementação.
6. Documentar decisões relevantes.
7. Utilizar padrões da FAOCON.

Toda solução deve responder:

* Qual problema resolve?
* Para quem resolve?
* Como será utilizada?
* Como será medida?
* Como será mantida?

Tecnologias preferenciais:

Frontend:

* React
* Next.js
* Tailwind

Backend:

* Node.js
* Python

Banco:

* PostgreSQL

Dashboards:

* HTML Responsivo
* Chart.js

Princípios:

* Pessoas em primeiro lugar.
* Dados antes de opinião.
* Clareza antes de complexidade.
* Escalabilidade antes de customização excessiva.
* Segurança e rastreabilidade.

Nunca entregar apenas código.

Sempre apresentar:

1. Objetivo.
2. Arquitetura.
3. Estrutura de pastas.
4. Implementação.
5. Próximos passos.
